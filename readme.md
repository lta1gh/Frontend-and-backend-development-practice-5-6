# Отчет по выполнению практических работ, включенных в блок "Контрольная работа 1"

## Состав репозитория
В данном репозитории представлены решения всех практических работ (п.р. 1-6), по дисциплине "Фронтенд и бэкенд разработка", относящиеся к блоку "Контрольная работа 1". 

## Практическое задание 1
1. В ходе выполнения работы был установлен препроцессор LESS для упрощения и ускорения работы над стилизацией веб-приложения.
2. После изучения синтаксиса LESS и применения его на практике для создания дизайна карточки товара (Общие параметры карточки, реакция при наведении курсора, расположение изображения, стиль заголовка и описания, стиль кнопки и ее реакция при наведении курсора), файл styles.less был скомпилирован в файл styles.css для использования его в HTML. 
3. После компиляции css файла был создан HTML файл, задействующий его. В результате была получена карточка товара, демонстрирующая ранее описаные измения дизайна.

## Практическое задание 2
1. В ходе выполнения практического задания 2 был установлен express.js для создания приложения. 
2. В файле app.js было написано приложение для управления доступными услугами. Каждая услуга включала в себя ID, название и цену. Также были реализованны все CRUD (Create, Read, Update, Delete)
3. После запуска командой `node app.js` сервер был активен и мог принимать запросы CRUD.
4. Для тестирования приложению отправлялись запросы curl через командную строку. В ходе тестирования удалось получить список всех доступных товаров, получить конкретный товар, создать новый товар, изменить товар и удалить товар.

## Практическое задание 3
1. Для тестирования приложения из практического задания 2 был установлен Postman.
2. В самой программе было установлено соединение с активным сервером приложения.
3. При помощи интерфейса Postman были созданы несколько запросов для тестирования приложения, среди которых: получение всех товаров, создание товара и его удаление.
4. Для openweathermap был получен ключ API для тестирования запросов к погоде.
5. После ввода ключа и города и типа запроса были получены как погода на момент обращения, так и прогнозы погоды на 5 дней.

## Практическое задание 4
1. Для связки API из предыдущих практик и интерфейса, написанного на React были установлены необходимые зависимости в папку backend (express, nanoid), а также дополнен и изменен код (добавлен middleware для логирования запросов, обработчик ошибок и проверки корректности данных в запросах, заменены численные ID на nanoid).
2. Для работы с API был написан frontend, а также стиль на scss для последующей конвертации в css
3. После установки axios в папке frontend был создан index.js для подключения бэкенда к фронтенду, а также изменен ServicesPage.jsx.
4. Была установлена библиотека cors и внесены соответствующие упоминания в серверную часть.
5. После запуска клиента и сервера был успешно отображен графический интерфейс, позволяющий выполнять все CRUD операции без написания кода.

## Практическое заданние 5
1. В backend часть были установлены необходимые зависимости и swagger.
2. Был создан js файл, описывающий API для управления услугами с полной документацией.
3. При запуске сервера была доступна ранее созданная интерактивная документация, а при помощи "Try it out" была протестированна работоспособность функции получения товара по ID.
